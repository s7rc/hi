## Technical Details

**The Fix:**
Changed joystick offset calculation from fixed `0.25f` to dynamic:
```kotlin
val maxOffsetFraction = (1f - foregroundSize) / 2f
```

This ensures the inner circle movement scales based on the available ring space between inner and outer circles.

**Example behavior:**
- `foregroundSize = 0.66` → `maxOffsetFraction = 0.17` ✓
- `foregroundSize = 0.5` → `maxOffsetFraction = 0.25` ✓
- Smaller inner circle = more movement range
